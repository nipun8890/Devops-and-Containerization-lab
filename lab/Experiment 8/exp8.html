Theory

Problem Statement

Managing infrastructure manually across multiple servers leads to configuration drift, inconsistent environments, and repetitive work.

What is Ansible?

Ansible is an open-source automation tool used for configuration management, application deployment, and orchestration.

  • Agentless (SSH based)
  • YAML Playbooks

Key Concepts

ComponentDescription
Control NodeWhere Ansible runs
Managed NodesTarget systems
InventoryList of servers
PlaybooksAutomation files
TasksActions
ModulesBuilt-in tools
RolesReusable structure

Installation

pip install ansible ansible --version sudo apt update -y sudo apt install ansible -y

Test Command

ansible localhost -m ping

Docker + SSH Setup

docker build -t ubuntu-server . docker run -d -p 2222:22 ubuntu-server ssh root@localhost -p 2222

Ansible Inventory

[servers] 172.17.0.2 172.17.0.3 172.17.0.4 [servers:vars] ansible_user=root

Playbook

- name: Configure servers hosts: all become: yes tasks: - name: Install packages apt: name: vim state: present

Conclusion

Ansible simplifies infrastructure management through automation. It ensures consistency, scalability, and faster deployment.
