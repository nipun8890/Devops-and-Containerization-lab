Objective

Perform static code analysis using SonarQube to detect bugs, vulnerabilities, code smells, and maintainability issues.

Theory

Problem Statement

Code issues are often discovered too late — during testing or after deployment. Manual code reviews are slow, not scalable, and may miss critical issues.

What is SonarQube?

SonarQube is an open-source platform for static code analysis. It analyzes source code without executing it and detects bugs, vulnerabilities, and code smells.

Key Terms

TermMeaning
Quality GateRules code must pass before deployment
BugCode that may break
VulnerabilitySecurity weakness
Code SmellPoor quality code
Technical DebtTime required to fix issues
CoveragePercentage of tested code
DuplicationRepeated code

Architecture

Your Code → Sonar Scanner → SonarQube Server → PostgreSQL Database

Prerequisites

  • Docker installed
  • Docker Compose installed
  • Terminal access
docker --version docker-compose --version

Steps

Step 1: Start SonarQube

cd ~/Desktop/exp10 docker-compose up -d docker-compose logs -f sonarqube docker ps

Step 2: Open Dashboard

http://localhost:9000 username: admin password: admin

Step 3: Generate Token

export SONAR_TOKEN=your_token_here

Step 4: Run Analysis

sonar-scanner \ -Dsonar.projectKey=my-app \ -Dsonar.sources=. \ -Dsonar.host.url=http://localhost:9000 \ -Dsonar.token=$SONAR_TOKEN

Step 5: View Results

http://localhost:9000/dashboard?id=my-app

Common Errors & Fixes

# Port already in use docker stop <container_id> # Node.js not found sudo apt install nodejs npm -y # Network issue docker-compose down docker-compose up -d

Jenkins Integration

pipeline { agent any stages { stage('SonarQube Analysis') { steps { sh 'mvn sonar:sonar' } } } }

Conclusion

SonarQube improves code quality by detecting issues early, enhancing security, and automating code review in CI/CD pipelines.
