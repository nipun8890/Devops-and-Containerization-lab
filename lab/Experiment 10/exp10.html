Theory

Problem Statement

Running containers manually works for simple applications, but real-world applications require scaling, reliability, and automation.

What is Orchestration?

Orchestration is the automatic management of containers including scaling, self-healing, load balancing, and service management.

Progression Path

docker run → Docker Compose → Docker Swarm → Kubernetes
LevelDescription
docker runSingle container
ComposeMulti-container (single machine)
SwarmBasic orchestration
KubernetesAdvanced orchestration

Compose vs Swarm

FeatureDocker ComposeDocker Swarm
ScopeSingle machineCluster
ScalingLimitedBuilt-in
Self-healingNoYes
Load balancingNoYes
Use caseDevelopmentProduction

Prerequisites

  • Docker installed
  • Docker Compose installed
  • Swarm mode enabled
  • Experiment 6 setup

Practical Steps

Step 1: Stop Existing Containers

docker compose down -v docker ps

Step 2: Initialize Swarm

docker swarm init docker node ls

Step 3: Deploy Stack

docker stack deploy -c docker-compose.yml wpstack

Step 4: Verify Deployment

docker service ls docker service ps wpstack_wordpress docker ps

Step 5: Access Application

http://localhost:8080

Step 6: Scale Application

docker service scale wpstack_wordpress=3

Step 7: Test Self-Healing

docker ps | grep wordpress docker kill <container-id> docker service ps wpstack_wordpress

Step 8: Remove Stack

docker stack rm wpstack docker service ls docker ps

Analysis

  • Same Compose file works in Swarm
  • Swarm uses services instead of containers
  • Scaling is automatic
  • Self-healing replaces failed containers
  • Load balancing is handled internally

Conclusion

Docker Swarm extends Docker Compose by adding orchestration, scaling, and self-healing capabilities. It acts as a bridge between basic container usage and advanced orchestration tools like Kubernetes.
